Features

Real-time monitoring

WebSocket-powered updates every 2 seconds. See CPU, memory, uptime, and active/idle status for every Claude session without refreshing.

Context window tracking

Each card shows a progress bar of context token usage against the 200k window. Green, amber, and red thresholds so you know when a session is running hot.

Agentboard integration

Projects with an .agentboard folder show a full kanban view in the detail panel with backlog, todo, in-progress, and done columns.

Git status at a glance

See the current branch, last commit, dirty file count, and remote URL for each project without switching terminals.

Expandable detail panel

Click any card to open a resizable split-screen with tabs for overview, git, agentboard, package.json, and CLAUDE.md configuration.

Sort lock

Lock the card order so sessions stop jumping around when CPU fluctuates. New sessions appear at the end. State persists across page reloads.

How it works

1

Process discovery

Parses ps aux to find all running claude CLI processes on your machine.

2

Project resolution

Uses lsof to inspect each process's open files and infer the project working directory.

3

Context tracking

Reads the latest assistant message from .claude/projects/ session files to extract token usage data.

4

Enrichment

On card click, fetches git info, package.json, CLAUDE.md, and agentboard data from the project directory.

5

Live broadcast

Bun's built-in WebSocket server pushes session data to all connected browsers every 2 seconds.

Traffic Control is a local-only tool for macOS. Requires Bun v1.0+.
